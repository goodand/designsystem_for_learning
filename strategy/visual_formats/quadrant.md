# Quadrant / Contrast Table Format

## Use When

- concept positioning by two axes
- strategy comparison
- lexical vs semantic with another axis

## Formats

- 2-column contrast table for binary comparison
- 2x2 quadrant for two independent axes
- lane layout for more than two categories

## Tool Preference

- Markdown table for text-first explanation
- SVG/HTML for quadrant maps with positioned concept cards

## UX Rules

- Name both axes before placing cards.
- Place one concept card at a time.
- Do not use more than two axes in one quadrant.

## Avoid

- Do not force non-MECE categories into a quadrant.
