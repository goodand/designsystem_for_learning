# One-Point Animation Format

## Use When

- one change is the lesson
- vector normalization
- row pair selection
- query entering retriever
- hidden state update

## Beat Pattern

```text
show stable scene
-> highlight one object
-> perform one change
-> pause
-> label what changed
```

## Tool Preference

- CSS/SVG animation for lightweight motion
- Canvas for many moving points
- slide animation for storyboard/prototype

## UX Rules

- One object changes per beat.
- Do not animate transformation and classification at the same time.
- Use short pauses after the key change.

## Avoid

- Avoid decorative motion that does not encode meaning.
